## 🚀 Быстрый старт (шаг за шагом)

### Шаг 1: Установи Node.js

Скачай и установи с https://nodejs.org (версия 18 или выше)

Проверь установку:
```bash
node --version   # должно быть v18+
npm --version
```

### Шаг 2: Положи папку проекта

Скопируй папку `quizapp` к себе на компьютер, например в `C:\Users\ИМЯ\Desktop\quizapp`

### Шаг 3: Запусти бэкенд

Открой **первый** терминал (cmd / PowerShell / Terminal):

```bash
cd quizapp/backend
npm install
npm run dev
```

✅ Ты должен увидеть: `✅ Сервер запущен на http://localhost:3001`

### Шаг 4: Запусти фронтенд

Открой **второй** терминал (не закрывая первый!):

```bash
cd quizapp/frontend
npm install
npm run dev
```

✅ Ты должен увидеть: `Local: http://localhost:3000/`

### Шаг 5: Открой приложение

Зайди в браузер: **http://localhost:3000**

## 🎮 Как пользоваться приложением

### Для организатора:

1. **Регистрация** → выбери роль "Организатор"
2. **Создать квиз** → нажми "Создать квиз" в меню
3. **Заполни настройки**: название, категория, время на вопрос
4. **Добавь вопросы**: нажми "Добавить вопрос", введи текст, варианты ответов, отметь правильный (нажав на букву), сохрани
5. **Запусти квиз**: нажми "▶️ Запустить"
6. **Поделись кодом комнаты** — участники видят его на экране (6 букв/цифр)
7. Нажми **"Начать квиз"** когда все подключились
8. **Управляй вопросами**: нажимай "Показать ответы" → "Следующий вопрос"

### Для участника:

1. **Регистрация** → выбери роль "Участник"
2. Нажми **"Войти в игру"** в меню
3. Введи **код комнаты** от организатора
4. Жди начала квиза
5. Выбирай ответы и нажимай **"Подтвердить"**
6. После каждого вопроса видишь промежуточный рейтинг

## 🛠️ Возможные проблемы

### "npm install" не работает
```bash
# Попробуй очистить кэш:
npm cache clean --force
npm install
```

### Порт уже занят (EADDRINUSE)
```bash
# Windows — найди и убей процесс:
netstat -ano | findstr :3001
taskkill /PID <номер_pid> /F

# Mac/Linux:
lsof -ti:3001 | xargs kill
```

### WebSocket не подключается
- Убедись что бэкенд запущен (`http://localhost:3001/api/health` должен вернуть `{"ok":true}`)
- Не используй разные порты в vite.config.js и server.js

### Ошибка "better-sqlite3"
```bash
# Если Windows и ошибка компиляции:
npm install --global windows-build-tools
# или:
npm install better-sqlite3 --build-from-source
```

## 🌐 Деплой (для публичного доступа)

### Вариант 1: Railway (бесплатно)
1. Зайди на https://railway.app
2. Подключи GitHub репозиторий
3. Добавь бэкенд как сервис, фронтенд как статику

### Вариант 2: Render (бесплатно)
1. https://render.com
2. New → Web Service → укажи папку backend
3. Build command: `npm install`
4. Start command: `node server.js`

### Вариант 3: Локальная сеть
Если все в одной сети (Wi-Fi), замени в `vite.config.js`:
```js
proxy: {
  '/api': 'http://192.168.x.x:3001',  // ← IP твоего компьютера
```
И участники заходят на `http://192.168.x.x:3000`

## 📊 Технологии

| Что | Чем сделано |

## 🗃️ База данных

SQLite файл создаётся автоматически в `backend/data/quiz.db`

Таблицы:
- **users** — пользователи (участники и организаторы)
- **quizzes** — квизы с настройками и кодами комнат
- **questions** — вопросы (текст, картинки, варианты ответа)
- **quiz_sessions** — сессии участия (итоговые баллы, места)
- **answers** — ответы на каждый вопрос
