## 🎮 Как пользоваться приложением

### Для организатора:

1. **Регистрация** → выбери роль "Организатор"
2. **Создать квиз** → нажми "Создать квиз" в меню
3. **Заполни настройки**: название, категория, время на вопрос
4. **Добавь вопросы**: нажми "Добавить вопрос", введи текст, варианты ответов, отметь правильный (нажав на букву), сохрани
5. **Запусти квиз**: нажми " Запустить"
6. **Поделись кодом комнаты** — участники видят его на экране (6 букв/цифр)
7. Нажми **"Начать квиз"** когда все подключились
8. **Управляй вопросами**: нажимай "Показать ответы" → "Следующий вопрос"

### Для участника:

1. **Регистрация** → выбери роль "Участник"
2. Нажми **"Войти в игру"** в меню
3. Введи **код комнаты** от организатора
4. Жди начала квиза
5. Выбирай ответы и нажимай **"Подтвердить"**
6. После каждого вопроса видишь промежуточный рейтинг

### Описание кода 

### Backend
backend/server.js

Главный файл сервера. Запускает Express-приложение и HTTP-сервер, подключает все маршруты (авторизация, квизы, загрузка файлов), инициализирует WebSocket-сервер на пути /ws. При подключении по WebSocket проверяет JWT-токен пользователя и передаёт соединение в менеджер комнат. Точка входа всего бэкенда.

backend/db/database.js

Инициализация и управление базой данных SQLite через библиотеку sql.js. При первом запуске создаёт файл quiz.db и все таблицы: users, quizzes, questions, quiz_sessions, answers. Экспортирует обёртку над SQL-запросами (prepare) с методами run, get, all — аналог синхронного API для удобной работы с БД. После каждого изменения автоматически сохраняет файл на диск.

backend/middleware/auth.js

Middleware для защиты маршрутов. Содержит две функции: authMiddleware — проверяет наличие и валидность JWT-токена в заголовке запроса, добавляет данные пользователя в req.user; organizerOnly — дополнительная проверка роли, запрещает доступ всем кроме организаторов. Используется во всех защищённых эндпоинтах API.

backend/routes/auth.js

Маршруты авторизации. Три эндпоинта: POST /register — регистрация нового пользователя с хешированием пароля через bcryptjs, выдаёт JWT-токен; POST /login — вход по email и паролю, возвращает токен; GET /me — возвращает данные текущего авторизованного пользователя.

backend/routes/quizzes.js

Маршруты для работы с квизами и вопросами. Реализует полный CRUD для квизов (создание, просмотр, редактирование, удаление), управление вопросами внутри квиза, получение лидерборда по итогам, историю участия пользователя, поиск квиза по коду комнаты. Все изменяющие операции доступны только организаторам.

backend/routes/upload.js

Маршрут для загрузки изображений к вопросам. Принимает файл через multipart/form-data, проверяет что это изображение и что размер не превышает 5 МБ, сохраняет на диск в папку data/uploads под уникальным именем. Возвращает URL, который затем сохраняется в вопросе.

backend/wsManager.js

Ядро реального времени. Управляет комнатами квизов через WebSocket. Обрабатывает все события: join_room — подключение организатора или участника к комнате; start_quiz — запуск квиза; next_question — переход к следующему вопросу с рассылкой всем участникам; end_question — завершение вопроса и показ правильных ответов; submit_answer — приём ответа от участника с подсчётом очков (учитывается скорость ответа); force_end_quiz — принудительное завершение. Ведёт лидерборд в реальном времени.

 ### Frontend
frontend/src/context/AuthContext.jsx

Глобальный контекст авторизации. Хранит данные текущего пользователя и JWT-токен, предоставляет функции login, register, logout и универсальный apiFetch для запросов к API с автоматической подстановкой токена. При старте приложения восстанавливает сессию из localStorage.

frontend/src/hooks/useWebSocket.js

Хук для работы с WebSocket-соединением. Предоставляет функции connect (открыть соединение с передачей токена через URL-параметр), send (отправить JSON-сообщение) и disconnect (закрыть соединение). Автоматически закрывает соединение при размонтировании компонента.

frontend/src/App.jsx

Корневой компонент приложения. Настраивает роутинг через React Router: определяет все страницы и их URL, оборачивает защищённые маршруты в компонент RequireAuth, который проверяет авторизацию и роль пользователя перед рендером страницы.

frontend/src/pages/Login.jsx и Register.jsx

Страницы входа и регистрации. Register дополнительно содержит визуальный выбор роли (участник / организатор) через карточки с иконками. После успешной авторизации перенаправляют пользователя на нужную страницу в зависимости от роли.

frontend/src/pages/Dashboard.jsx

Личный кабинет организатора. Отображает список всех созданных квизов с кодами комнат, статусами и количеством участников. Позволяет перейти к редактированию, запуску или просмотру результатов каждого квиза, а также удалить его.

frontend/src/pages/QuizEditor.jsx

Редактор квизов. Позволяет задать настройки (название, категория, время на вопрос) и управлять списком вопросов. Для каждого вопроса можно выбрать тип (один или несколько правильных ответов), добавить текст, загрузить изображение, настроить варианты ответов и правильный ответ. Вопросы сохраняются на сервер по одному.

frontend/src/pages/HostQuiz.jsx

Панель управления квизом для организатора в реальном времени. Показывает зал ожидания с подключившимися участниками, код комнаты, кнопку запуска. Во время квиза отображает текущий вопрос, обратный таймер, счётчик ответивших участников, кнопки перехода к следующему вопросу и завершения. После каждого вопроса показывает промежуточный лидерборд.

frontend/src/pages/JoinQuiz.jsx

Страница входа в квиз для участника. Содержит поле ввода 6-значного кода комнаты. После ввода проверяет существование квиза через API и перенаправляет на страницу игры.

frontend/src/pages/PlayQuiz.jsx

Игровой экран участника. Подключается к комнате через WebSocket и реагирует на все события от организатора: отображает вопрос с вариантами ответов и таймером, принимает выбор пользователя, отправляет ответ и показывает результат (правильно/неправильно, заработанные очки). После каждого вопроса показывает промежуточный рейтинг, в конце — итоговый лидерборд.

frontend/src/pages/Profile.jsx

Личный кабинет участника. Показывает аватар с инициалом, статистику (количество пройденных квизов, суммарные очки, количество побед) и полную историю участия с местами и баллами за каждый квиз.

frontend/src/index.css

Глобальные стили приложения. Определяет дизайн-систему через CSS-переменные (цвета, скругления, тени), стили базовых компонентов (кнопки, карточки, поля ввода, бейджи, лидерборд, таймер), адаптивную сетку для разных размеров экрана.
